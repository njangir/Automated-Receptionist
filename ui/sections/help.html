Common Issues

Server Won't Start

  • Check if the port is already in use by another application.
  • Verify that all required dependencies are installed.
  • Check the log files for error messages.

Chrome Not Starting

  • Ensure Google Chrome is installed on your system.
  • Check the Chrome executable path in configuration.
  • Verify that the Chrome debug port (default: 9222) is not in use.
  • Try manually starting Chrome with remote debugging enabled.

Configuration Not Saving

  • Check file permissions for the .env.config file.
  • Ensure the application has write access to the configuration directory.
  • Verify that the configuration values are valid (check validation errors).

Agent Not Starting

  • Verify the agent entrypoint file exists in the agents directory.
  • Check that all required environment variables are set in .env.secrets.
  • Review agent logs for specific error messages.
  • Ensure LiveKit server is running and accessible.

Troubleshooting Steps

  1. Review the error messages in the UI or log files.
  2. Verify all configuration values are correct.
  3. Check that all required services (Chrome, LiveKit) are running.
  4. Restart the application and try again.
  5. Check system resources (memory, disk space).

Getting More Help

If you continue to experience issues:

  • Check the application logs in the configured log directory.
  • Review the error messages carefully - they often contain helpful information.
  • Ensure you're using the latest version of the application.
  • Verify your system meets the minimum requirements (Python 3.9+, Chrome browser).

System Requirements

  • Python 3.9 or higher
  • Google Chrome browser (for browser automation)
  • Sufficient disk space for logs and temporary files
  • Network access for API calls and LiveKit connections
