Getting Started

Welcome to the MyAgent Voice Agent Server. This application manages voice AI agents that handle phone calls using LiveKit. Follow these steps to set up and use the application:

Initial Setup

  1. Sign In: When you first open the application, you'll be prompted to sign in with your Firebase credentials. Only existing accounts can sign in (sign up is disabled).
  2. Configure Settings: Navigate to the Configuration tab to set up your server settings, Chrome launcher options, agent paths, and other preferences.
  3. Review Defaults: Most settings have sensible defaults. Only modify what you need to customize.
  4. Save Configuration: Click "Save Configuration" to save your settings. The server is already running, so changes take effect immediately or may require a restart depending on the setting.
  5. Install Modules (Optional): If you need browser automation modules, go to the Modules tab, sign in if needed, and install required files from Firebase.
  6. Configure Performance Profile: Set the performance profile based on your system capabilities (low for low-end PCs, balanced for normal use, high for best quality).
  7. Turn Detector Models (Optional): Turn detector models are completely optional. The agent works fine without them. If you want better conversation flow, models can be downloaded automatically when needed. See the "Turn Detector Model Installation" section below for details.

Application Tabs

  • Dashboard: Monitor agent status, control agent operations, view statistics, and manage webhook listening.
  • Configuration: Configure all application settings organized by category.
  • Call History: View past calls, search and filter call records, and read full transcripts.
  • Modules: Download and install browser automation modules from Firebase (requires authentication).
  • Instructions: This guide - getting started and usage information.
  • Help & Support: Troubleshooting guides, common issues, and contact information.
  • About: Version information and build details.
  • Credits: Acknowledgments and credits.

Configuration Categories

  • Server: Configure the host and port where the FastAPI server will run (default: 0.0.0.0:8000).
  • Process Management: Settings for agent process management, timeouts, and startup delays.
  • Agent Paths: Configure paths to your agent files and log directories.
  • Google Sheets: Set up Google Sheets integration for contact lookup during incoming calls.
  • Chrome Launcher: Configure Chrome browser settings for automation (debug port, user data directory, executable path).
  • Browser Automation: Settings for backoffice login URL, login type, and automation behavior.
  • Agent Configuration: Default agent use cases, audio device selection, and agent behavior settings.
  • Performance: Configure performance profiles and optimization settings for low-end or high-end systems.
  • Logging: Log directory paths and log management settings.

Using the Dashboard

The Dashboard is your main control center:

  • Server Status: Shows if the main server is running.
  • Webhook Toggle: Enable/disable webhook listening. When enabled, Chrome starts automatically, browser service connects, and login is executed. When disabled, agent stops, browser service closes, and Chrome stops.
  • Agent Type: Choose between "Bundled" (uses myagent.py) or "Online" (fetches agent code from Firebase).
  • Statistics: View total calls, active calls, agent uptime, and average rating.
  • Agent Status: Monitor agent process status, PID, uptime, and call status.
  • Controls:
    • Start Agent: Manually start an agent with client code, phone number, and name.
    • Start Test Agent: Quick start with test credentials.
    • Stop Agent: Gracefully stop the running agent.
    • Manual Overtake: Stop agent but keep call alive for manual intervention.
    • End Call: Stop agent and end the call.

Webhook Workflow

When webhook listening is enabled:

  1. Incoming calls trigger the webhook endpoint with a phone number.
  2. The system looks up the contact in Google Sheets using the phone number.
  3. If found, the system makes a pick request to the configured pick service.
  4. The voice agent starts automatically with the contact information.
  5. Call transcripts and summaries are saved to the call history.

Call History

The Call History tab allows you to:

  • View all past calls with details (date, time, duration, client info, summary, mood, rating).
  • Search calls by phone number, client name, client code, or summary.
  • Filter calls by date range.
  • View full transcripts with speaker identification and timestamps.
  • Navigate through paginated results.

Performance Profiles

The application supports three performance profiles to optimize resource usage based on your system capabilities:

  • Low Profile: Optimized for low-end Windows PCs and systems with limited resources.
    • Uses faster STT model (nova-2) for reduced CPU usage
    • Limits LLM response length (100 tokens) for faster generation
    • Disables preemptive generation to reduce CPU load
    • Disables noise cancellation to save processing power
    • Skips turn detection (models not required)
    • Expected impact: 30-50% CPU reduction, 20-30% memory reduction
  • Balanced Profile (Default): Recommended for most systems with moderate resources.
    • Uses standard STT model (nova-3) for good accuracy
    • Moderate LLM response length (150 tokens)
    • Enables preemptive generation for better responsiveness
    • Enables noise cancellation for better audio quality
    • Turn detection optional (works without models, better with models)
  • High Profile: Best quality for high-end systems with ample resources.
    • Uses standard STT model (nova-3) for best accuracy
    • Longer LLM response length (200 tokens) for detailed responses
    • Enables preemptive generation for optimal responsiveness
    • Enables noise cancellation for best audio quality
    • Turn detection recommended (better conversation flow)

Configuration: Set PERFORMANCE_PROFILE in the Configuration tab under the Performance category. You can also override individual settings (STT model, LLM tokens, etc.) if needed.

Turn Detector Model Installation

Turn detection improves conversation flow by detecting when the user has finished speaking. The agent works perfectly fine without turn detector models - they are completely optional.

Automatic Behavior: The agent automatically skips turn detection if models are not found, regardless of performance profile. If models are available, they will be used for better conversation flow in balanced and high profiles.

Downloading Turn Detector Models (Optional)

If you want to enable turn detection, you can download the models using LiveKit's CLI tools. The models will be automatically downloaded to your system's cache directory (managed by LiveKit) when first needed, or you can download them manually:

What gets downloaded: The following files are stored in your system's cache directory:

  • model_q8.onnx - The quantized ONNX model for turn detection
  • languages.json - Language configuration file

Verification: When you start the agent, check the logs. If models are found, you'll see "Turn detector initialized successfully". If models are missing, you'll see a warning message, but the agent will continue to work normally without turn detection.

Note: The agent is designed to work without turn detection models. They are an optional enhancement for better conversation flow, but not required for the agent to function.

Module Management

The Modules tab enables you to:

  • Sign in with Firebase credentials (required for downloading modules).
  • View your installer ID (used to identify which modules are available for your installation).
  • Browse available modules from Firebase.
  • Install required browser automation modules.
  • View installed modules and their versions.

Important Notes

  • Authentication: You must sign in to use the application. Authentication is required for module downloads and is validated on startup.
  • Chrome Browser: Ensure Chrome is installed for browser automation features. Chrome will auto-start when webhook listening is enabled.
  • Security: Keep your .env.secrets file secure - it contains sensitive credentials like API keys and Firebase configuration.
  • File Paths: Use absolute paths for file paths to ensure cross-platform compatibility.
  • Ports: Ensure port numbers are available and not in use by other applications (default server port: 8000, Chrome debug port: 9222).
  • Agent Types: "Bundled" uses the local myagent.py agent file. "Online" fetches agent code from Firebase based on your installer ID.
  • Configuration Changes: Some configuration changes require restarting services. The application will prompt you when restarts are needed.
  • Performance Profiles: For low-end Windows PCs, use the "low" performance profile to reduce CPU and memory usage. Turn detector models are automatically skipped if not found, regardless of performance profile.
  • Turn Detector Models: Turn detector models are completely optional. The agent automatically skips turn detection if models are not found, regardless of performance profile. Models improve conversation flow but are not required for the agent to function.
  • Logs: All logs are stored in the configured log directory. Log cleanup runs automatically to manage disk space.

Closing the Application

Use the "Close App" button in the header to shut down the application. This will:

  • Stop any running agent processes
  • Close browser service connections
  • Stop Chrome instances
  • Shut down the API server

Warning: This action cannot be undone. Make sure to save any important work before closing.
